Title: "Implement Authentication & Authorization System using Clerk + Cloudflare Access"\n\nBody:\n## Overview\n\nImplement a full authentication, authorization, and user registration system for adblock-compiler. Based on architectural review of the existing stack (Cloudflare Workers, Angular SSR, D1, KV, R2, Hyperdrive вҶ’ PlanetScale, Durable Objects, Queues, Workflows, Turnstile), the recommended approach is **Clerk** for user auth/registration + **Cloudflare Access** for admin protection.\n\n> **Do not wait for the PlanetScale backend to be fully built out.** Auth is JWT-based and completely database-agnostic. The D1/PlanetScale backend only stores a `clerk_user_id` FK вҖ” the auth flow itself never touches your database.\n\n---\n\n## Technology Decisions\n\n| Layer | Technology | Rationale |\n|---|---|---|\n| **User auth + registration** | [Clerk](https://clerk.com) | Edge-native JWT, `@clerk/angular` SDK, managed user DB, social OAuth, passwordless, passkeys |\n| **Admin access** | [Cloudflare Access](https://developers.cloudflare.com/cloudflare-one/applications/) | Zero-code network-layer protection for `/admin` routes, integrates with Google/GitHub/Okta |\n| **Bot protection** | Cloudflare Turnstile | Already deployed вҖ” covers unauthenticated endpoints |\n| **Authorization (roles)** | Clerk custom JWT claims + D1 roles table | Simple FK relationship |\n| **API key auth** | Custom вҖ” KV + D1 | Hash & store API keys in KV, associate with `clerk_user_id` |\n\n### Why Clerk over Auth0 / WorkOS?\n- **Edge-native**: JWT verification uses `crypto.subtle` вҖ” no Node.js shim, no roundtrip, works natively in Cloudflare Workers\n- **Angular SDK**: `@clerk/angular` ships drop-in `<SignIn>`, `<UserButton>` components and `AuthGuard` for Angular Router\n- **No registration system to build**: Clerk handles sign-up, email verification, password reset, social OAuth, magic links, and passkeys out of the box\n- **Generous free tier**: 10,000 MAU free\n- **Database-agnostic**: Works today with D1, works tomorrow with PlanetScale вҖ” zero auth changes required when the DB is swapped\n\n---\n\n## |\n|---|---|---|\n| **Anonymous** | None | Compile up to N lists/hour (rate-limited by Turnstile + IP via `RATE_LIMIT` KV) |\n| **Free registered** | Clerk sign-up | Higher rate limits, saved lists in using Clerk's JWKS endpoint + `crypto.subtle` (no external deps required):\n  ```typescript\n  import { createRemoteJWKSet, jwtVerify } from 'jose';\n  const JWKS = createRemoteJWKSet(new URL('https://<clerk-domain>/.well-known/jwks.json'));\n  async function verifyAuth(request: Request): Promise<string | null> {\n    const token = request.headers.get('Authorization')?.replace('Bearer ', '');\n    if (!token) return null;\n    try {\n      const { payload } = await jwtVerify(token, JWKS);\n      return payload.sub as string; // clerk_user_id\n    } catch { return null; }\n  }\n  ```\n- [ ] Gate the following routes with auth middleware: `POST /compile`, `GET/POST/DELETE /lists/*`, `GET /admin/*`\n- [ ] Return `401 Unauthorized` for unauthenticated requests to protected
